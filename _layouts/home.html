   The rapid growth of OpenHarmony has driven the need to port JavaScript and TypeScript (JS/TS) libraries to ArkTS, a statically typed derivative for performance-critical mobile environments. However, most ports are one-off snapshots that quickly diverge from upstream evolution, leaving critical bug fixes and security patches unintegrated. Achieving repository-level co-evolution between JS/TS origins and ArkTS ports faces two major challenges: (1) massive, interdependent code changes between JS/TS versions that cause cascading compilation errors when integrated in batch; (2) the heavy structural refactoring of ArkTS (function-to-class conversion, added type declarations, re-implemented APIs) that makes it difficult to locate edit positions and ensure compatibility.
   We present ARKEVOLUTION, an LLM-augmented approach that enables incremental, repository-level co-evolution. To address the first challenge, ARKEVOLUTION constructs an Evolution-Aware Code Entity Dependency Graph (ECED) that uniformly models all code entities and their dependencies. It defines change-impact dependency to distinguish breaking from non-breaking changes, clusters entities with direct or indirect impact dependencies into groups. Each group is annotated with an LLM-generated change intent, allowing priority-based incremental or selective integration. For the second challenge, ARKEVOLUTION introduces a two-phase cross-language mapping: deterministic mapping via signature matching and inverse adaptation rules, followed by dependency-based propagation along the ECED graph. After localization, it integrates groups in ascending order of priority, retrieves repair strategies (e.g., building an adapter, preserving pre-change code, replacing reference names) from a pre-constructed experience knowledge base. Combining the change intent, JS/TS code diffs, the ArkTS context to be evolved, and the specific repair strategies, it guides the LLM to generate semantically equivalent and compatible code.
   We evaluate ARKEVOLUTION on 52 real-world upgrade tasks extracted from 13 ArkTS packages with explicit co-evolution histories, including 39 incremental and 13 leap upgrades. ARKEVOLUTION achieves 98.49% Precision, 99.51% Recall, and 99.00% F1-measure in identifying co-evolution-required entities. It attains a compilation pass rate of 90.14% and a test pass rate of 86.89%, significantly outperforming the state-of-the-art adaptation tool and ablation variants. In practical use, we upgraded eight popular ArkTS packages (e.g., @ohos/axios) with 21 evolution tasks, submitted pull requests, and 18 passed the full OpenHarmony CI toolchain quality review (ArkTS coding specifications, compilation, testing, and compatibility review). Of these, 16 (100%) have been merged by developers. This confirms the practical utility of our approach.
