   The rapid growth of OpenHarmony has driven the need to port JavaScript and TypeScript (JS/TS) libraries to ArkTS, a statically typed derivative for performance-critical mobile environments. However, most ports are one-off snapshots that quickly diverge from upstream evolution, leaving critical bug fixes and security patches unintegrated. Achieving repository-level co-evolution between JS/TS origins and ArkTS ports faces two major challenges: massive, interdependent code changes between JS/TS versions that cause cascading compilation errors when integrated in batch, and heavy structural refactoring of ArkTS, including function-to-class conversion, added type declarations, and re-implemented APIs, that makes it difficult to locate edit positions and ensure compatibility.
   We present ARKEVOLUTION, an LLM-augmented approach that enables incremental, repository-level co-evolution. To address large upstream change sets, ARKEVOLUTION constructs an Evolution-Aware Code Entity Dependency Graph (ECED) that uniformly models code entities and their dependencies. It distinguishes breaking from non-breaking changes through change-impact dependency, clusters entities with direct or indirect impact dependencies into groups, and annotates each group with an LLM-generated change intent for priority-based incremental or selective integration. To handle cross-language structural differences, ARKEVOLUTION uses deterministic mapping through signature matching and inverse adaptation rules, then propagates mappings along the ECED graph. It combines change intent, JS/TS diffs, ArkTS context, and repair strategies from an experience knowledge base to guide the LLM toward semantically equivalent and compatible ArkTS code.
   We evaluate ARKEVOLUTION on 52 real-world upgrade tasks extracted from 13 ArkTS packages with explicit co-evolution histories, including 39 incremental and 13 leap upgrades. ARKEVOLUTION achieves 98.49% Precision, 99.51% Recall, and 99.00% F1-measure in identifying co-evolution-required entities. It attains a 90.14% compilation pass rate and an 86.89% test pass rate, outperforming the state-of-the-art adaptation tool and ablation variants. In practical use, we upgraded eight popular ArkTS packages such as @ohos/axios with 21 evolution tasks, submitted pull requests, and 18 passed the full OpenHarmony CI toolchain quality review. The current public evaluation table marks 1 task as Merged, while the remaining tasks are under review.

Motivating Example: Axios
